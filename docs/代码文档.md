# 代码文档

1. 用户与权限：支持登录、注册、退出；角色包括普通用户和管理员。普通用户只能看到 active 论文，管理员可查看 pending、active、archived。
2. 文献检索：支持标题、摘要、DOI、作者、机构检索；支持年份精确搜索和 year_from/year_to 范围搜索；支持关键词多选筛选和收藏关键词筛选。
3. 文献入库：用户可手动填写论文信息，作者必须从作者表选择；关键词可自由输入并生成胶囊，保存时若不存在则自动写入 keyword 表。普通用户提交进入 pending，管理员提交默认为 active。
4. PDF 上传解析：单篇 PDF 上传后只预填表单，不直接入库；用户确认后提交。批量 PDF 解析会生成多篇待确认表单，确认后批量保存。
5. 管理员审核：管理员可将 pending 论文发布为 active，可将 active 论文归档为 archived，也可将 archived 重新发布为 active；管理员可删除论文。
6. 管理员修改：每篇论文后提供修改按钮，进入预填表单；修改后重新发布。修改页提供添加作者模块，新增作者后回到当前修改页。
7. 收藏功能：用户可收藏/取消收藏论文，收藏/取消收藏关键词；收藏页分为关键词区和论文区，收藏关键词可以筛选收藏论文。
8. 设置功能：支持中英文界面语言切换。
9. 数据导入导出：支持 CSV 导入；支持 JSON 和 CSV 导出。
10. 统计分析：首页显示论文数、作者数、关键词数、引用数；分析区展示年度分布和高频关键词。

## 接口说明

系统采用轻量级 Python Web 服务实现，主要文件为 app_mysql.py。模块划分如下：
- 配置模块：读取 MYSQL_HOST、MYSQL_PORT、MYSQL_USER、MYSQL_PASSWORD、MYSQL_DATABASE、MYSQL_CLI 等环境变量。
- 数据库访问模块：run_sql 使用 MySQL CLI 执行 SQL 并解析 TSV 输出；quote 负责 SQL 字符串转义。
- 数据校验与规范化模块：validate_paper_input 用于新增入库校验；normalize_paper_edit_input 用于管理员修改页保存；validate_date、validate_doi、ensure_unique_paper_doi、split_values 等处理常见字段，确保 DOI 非空唯一、作者来自已登记 ORCID 的作者表。
- 用户与权限模块：register_user、authenticate_user、current_user_id、current_user_role、cookie 登录状态维护。
- 文献业务模块：create_paper、update_paper、delete_paper、update_paper_status、paper_rows。
- 作者与关键词模块：create_author、author_options、keyword_options、favorite_keyword、unfavorite_keyword。
- 收藏与统计模块：favorite_paper、unfavorite_paper、user_favorites、stats。
- PDF 解析模块：upload_pdf、upload_pdfs 调用 services/pdf_parser.py，从 PDF 中提取标题、作者、摘要、DOI、关键词、来源等候选元数据。
- 页面渲染模块：render_login、render_register、render_index、render_paper_edit 生成 HTML。
- HTTP 路由模块：Handler.do_GET/do_POST 根据 URL 分发请求。
- 静态资源模块：static/style.css 定义平台视觉样式和响应式布局。
